MCP 是什么

你不需要研究原理，只要记住：MCP 就是 AI 和 Blockbench 之间的连接线。

01

Blockbench 开门

插件会在本机开启一个地址：localhost:3000/bb-mcp。

02

Trae 走这条线

Trae 通过 npx mcp-remote 连到这个地址。

03

强 AI 更重要

MCP 只是手。模型做得好不好，主要看 AI 会不会拆结构、控比例、安排骨骼。

MCP 配置步骤

步骤多，但不要跳。每张图点开都能看清大图。

01

从 URL 加载插件

Blockbench 里从 URL 加载插件，粘贴原作者提供的地址并确认。

02

下载 Trae 国际版

进入 Trae 官网，点击 Download SOLO Desktop 下载桌面端。

03

下载安装 Node.js

下载 Windows Installer，双击安装时一路点击下一步即可。

04

登录并切到 Code

打开 Trae 后先登录账号，再点击左上角切换到 Code 模式。

05

打开项目文件夹

点击选择文件夹，打开你的懒人包或模组项目目录。

06

验证三个命令

在终端输入 node -v、npm -v、npx -v，三个都输出版本号就成功。

07

进入 MCP 手动配置

打开设置里的 MCP 页面，点击添加并选择手动配置。

08

粘贴配置并确认

把 MCP JSON 全部粘贴进输入框，检查无误后点击确认。

09

看到绿色勾

blockbench 后面出现绿色勾就说明配置成功，失败就点重试。

10

重启后验证连接

建议先关闭并重新打开 Trae，再让它通过 MCP 放一个方块。

前置指令

先锁定 Minecraft 方块模型标准

推荐复制英文版。看不懂没关系，复制后在末尾写绘图需求，例如“做一个小蜗牛”。

推荐：英文版 
Role: Expert Minecraft model planning assistant for authentic vanilla-style Blockbench creation.
Objective: Plan and guide low-spec voxel models that strictly follow Minecraft's blocky aesthetic. Reject realism, smooth curves, high-detail surfaces, and modern polished shapes.
1. GEOMETRY AND SHAPE:
Build only with cuboids, blocks, flat planes, and chunky stair-step approximations. No spheres, cylinders, smooth curves, thin slants, or overly dense details. The object must be readable through simple block masses.
2. SCALE AND STRUCTURE:
Keep the model compact, low-spec, and easy to rebuild in Blockbench. Separate major parts into clear bones when needed, such as body, head, limbs, shell, tail, or accessories.
3. TEXTURE STYLE:
Use a restricted Minecraft-like palette. Prefer crisp pixel blocks, hard borders, and simple color regions. Avoid gradients, soft shading, blur, noise, and high-resolution texture details.
4. OUTPUT STANDARD:
Before making the final model, describe the main cuboids, their rough proportions, the bone hierarchy, and the simple animation idea if the model is a creature.

角色：专业的 Minecraft 原版风格模型规划助手，用于指导 Blockbench 制作低精度方块模型。
目标：规划和指导符合 Minecraft 方块美学的低精度体素模型。拒绝写实、光滑曲线、高细节表面和过度精致的现代造型。
1. 几何与形状：
只允许使用长方体、方块、平面和粗颗粒阶梯式近似形状。不要球体、圆柱体、光滑曲线、细斜面或过密细节。模型必须通过简单方块体块就能看懂。
2. 比例与结构：
模型要紧凑、低规格、方便在 Blockbench 里复现。必要时把主要部位拆成清晰骨骼，例如身体、头、四肢、外壳、尾巴或装饰。
3. 材质风格：
使用接近 Minecraft 的有限调色板。优先硬边界、清晰像素块和简单色块。避免渐变、柔光、模糊、噪点和高清材质细节。
4. 输出标准：
制作最终模型前，先说明主要长方体、粗略比例、骨骼层级；如果是生物，再给出一个简单动画想法。

更省事的路线

连上 MCP 之后，差距在稳定和成品

你可以先按本页把原作者 MCP 跑通。工具包模型板块处理的是后面的麻烦：更稳的连接、更强的任务约束、更少手动导出。

更强的模型前置指令框架，不用自己反复试长模板 独家的模型材质处理方案，模型不会一进游戏就廉价 增强 MCP：连接更稳，适配更多 AI 工具 支持 AI 自动导出模型文件、材质、动画，少做手工收尾
